아래는 사용자 관리 기능에 대한 요구사항, 데이터 흐름, 시나리오 및 흐름, 에러 처리, 테스트 케이스를 한 표로 합친 것입니다.

| **기능**                 | **설명**                                                                                                                                                           | **엔드포인트**         | **HTTP 메서드** | **요청 데이터**                                                                                                                                                                                                                                                                                           | **응답**                                                                                 | **데이터 흐름**                                                                                                                                                                                                                                                                                                         | **시나리오 및 흐름**                                                                                                                                                                                                                                                                                              | **에러 처리**                                                                                                                                                                    | **테스트 케이스**                                                                                   |
|--------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------------------|-----------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------|
| **회원가입**             | 사용자는 이메일, 비밀번호, 이름, 연락처 정보를 입력하여 회원가입할 수 있습니다.                                                                                     | `/api/signup`           | POST            | ```json { "email": "string", "password": "string", "name": "string", "contact": "string" } ```                                                                                                                                                                     | **200 OK**: 회원가입 성공<br>**400 Bad Request**: 입력 데이터 오류                      | 사용자가 입력한 정보를 서버로 전송 → 서버는 입력 정보 검증 후 DB에 저장 → 이메일 인증 요청을 생성하여 전송 → 사용자가 이메일 인증 링크를 클릭하여 계정을 활성화                                                                                                                                               | 사용자가 이메일, 비밀번호, 이름, 연락처 입력 후 '가입' 클릭 → 서버가 정보 검증 후 DB에 저장 → 이메일 인증 후 계정 활성화 → 로그인 페이지로 리다이렉트                                                                                                                                                               | **회원가입 에러**: 필수 필드 누락 시 적절한 오류 메시지 표시                                                                                                                                                     | **회원가입 테스트**: 올바른 형식의 이메일 및 비밀번호, 모든 필수 필드가 입력된 경우 성공                                                                 |
| **이메일 인증**          | 사용자는 이메일로 전송된 링크를 통해 계정을 활성화합니다.                                                                                                          | `/api/verify-email`     | GET             | 이메일 인증 토큰 (쿼리 파라미터로 전달)                                                                                                                                                                                                                          | **200 OK**: 인증 성공<br>**400 Bad Request**: 인증 실패                                 | 서버가 이메일 인증 요청을 생성하고 전송 → 사용자가 이메일 인증 링크를 클릭하여 계정을 활성화                                                                                                                                                                                  | 사용자가 이메일로 전송된 인증 링크를 클릭하여 계정을 활성화 → 계정 활성화 후 로그인 페이지로 리다이렉트                                                                                                                                                                        | **이메일 인증 에러**: 잘못된 인증 링크를 클릭할 경우 적절한 오류 메시지 표시                                                                                                                                      | **이메일 인증 테스트**: 올바른 인증 링크로 클릭 시 성공, 잘못된 링크 클릭 시 실패                                                    |
| **로그인**               | 사용자는 이메일과 비밀번호를 통해 로그인할 수 있습니다.                                                                                                            | `/api/login`            | POST            | ```json { "email": "string", "password": "string" } ```                                                                                                                                                                     | **200 OK**: 로그인 성공, JWT 반환<br>**401 Unauthorized**: 로그인 실패                  | 사용자가 입력한 로그인 정보 검증 → 세션 또는 JWT 생성                                                                                                                                                                                                                     | 사용자가 이메일과 비밀번호 입력 후 '로그인' 클릭 → 서버가 정보 검증 후 인증 토큰 발급 → 메인 페이지로 리다이렉트                                                                                                                                                               | **로그인 에러**: 이메일 또는 비밀번호 오류 시 '잘못된 로그인 정보입니다' 메시지 표시                                                                                                                                 | **로그인 테스트**: 올바른 이메일과 비밀번호 입력 시 성공, 잘못된 입력 시 실패                                                     |
| **로그아웃**             | 사용자는 로그아웃할 수 있습니다.                                                                                                                                   | `/api/logout`           | POST            | 없음                                                                                                                                                                                                                                                                                                     | **200 OK**: 로그아웃 성공                                                                | 사용자가 로그아웃 요청 → 세션 종료 또는 JWT 폐기                                                                                                                                                                                                                           | 사용자가 로그아웃 버튼 클릭 → 서버가 세션 종료 또는 JWT 폐기 → 로그아웃 후 로그인 페이지로 리다이렉트                                                                                                                                                                        | **로그아웃 에러**: 세션이 없거나 이미 종료된 경우 적절한 오류 메시지 표시                                                                                                                                         | **로그아웃 테스트**: 로그아웃 요청 시 세션 종료 성공                                                                                |
| **비밀번호 재설정**      | 사용자는 비밀번호를 잊은 경우 이메일을 통해 비밀번호를 재설정할 수 있습니다.                                                                                      | `/api/reset-password`   | POST            | ```json { "email": "string" } ```                                                                                                                                                                                                                                 | **200 OK**: 비밀번호 재설정 이메일 전송<br>**404 Not Found**: 이메일이 존재하지 않음    | 사용자가 이메일 입력 → 서버는 재설정 요청 생성 후 이메일 전송 → 사용자가 이메일 링크를 클릭하여 새 비밀번호 설정                                                                                                                                                            | 사용자가 이메일 입력 후 '재설정' 클릭 → 서버가 이메일 전송 → 사용자가 링크 클릭 후 새 비밀번호 설정 페이지로 이동                                                                                                                                                              | **비밀번호 재설정 에러**: 이메일이 존재하지 않을 경우 '이메일이 존재하지 않습니다' 메시지 표시                                                                                                                 | **비밀번호 재설정 테스트**: 존재하는 이메일 입력 시 성공, 존재하지 않는 이메일 입력 시 실패                                            |
| **사용자 정보 조회**      | 사용자는 자신의 계정 정보를 조회할 수 있습니다.                                                                                                                   | `/api/profile`          | GET             | 없음                                                                                                                                                                                                                                                                                                     | **200 OK**: 사용자 정보 반환                                                              | 사용자가 정보 조회 요청 → 서버가 DB에서 사용자 정보 조회 후 반환                                                                                                                                                                                                             | 사용자가 '내 정보' 페이지에서 정보 조회 요청 → 서버가 사용자 정보를 조회하여 페이지에 표시                                                                                                                                                                                    | **정보 조회 에러**: 인증되지 않은 사용자가 요청 시 '인증이 필요합니다' 메시지 표시                                                                                                                                 | **정보 조회 테스트**: 인증된 사용자가 요청 시 정보 조회 성공, 인증되지 않은 경우 실패                                                 |
| **사용자 정보 수정**      | 사용자는 자신의 이름, 연락처 정보를 수정할 수 있습니다.                                                                                                           | `/api/profile`          | PUT             | ```json { "name": "string", "contact": "string" } ```                                                                                                                                                                                                             | **200 OK**: 정보 수정 성공<br>**400 Bad Request**: 입력 데이터 오류                     | 사용자가 수정 요청을 서버로 전송 → 서버가 입력 정보 검증 후 DB 업데이트                                                                                                                                                                                                    | 사용자가 '내 정보 수정' 페이지에서 이름과 연락처 수정 후 '저장' 클릭 → 서버가 정보 검증 후 DB 업데이트 → 수정된 정보가 페이지에 반영                                                                                                                                             | **정보 수정 에러**: 필수 필드 누락 시 적절한 오류 메시지 표시                                                                                                                                                     | **정보 수정 테스트**: 올바른 형식의 이름과 연락처 입력 시 수정 성공, 필드 누락 시 실패                                                |
